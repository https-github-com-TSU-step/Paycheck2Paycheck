# Paycheck2Paycheck

**Paycheck2Paycheck** — это Android-приложение для людей с нерегулярным доходом (студентов, фрилансеров, молодых специалистов), которое помогает избавиться от финансовой неопределенности. Приложение автоматически рассчитывает безопасный дневной лимит трат до следующего поступления средств и пересчитывает его после каждой покупки.

## Технологический стек и архитектура

Проект построен на основе Многослойной архитектуры (Layered Architecture) согласно официальным рекомендациям Google для Android.

* **Язык:** Kotlin
* **UI:** Jetpack Compose
* **Архитектурный паттерн:** MVVM (Model-View-ViewModel)
* **Dependency Injection:** Dagger Hilt
* **Локальная БД:** Room
* **Слои приложения:**
  * `UI Layer`: Composable-функции и ViewModels.
  * `Domain Layer`: Бизнес-логика, Use Cases (расчёт дневного лимита) и модели предметной области (Budget, Expense, MoneyOperation).
  * `Data Layer`: Репозитории и источники данных (Room для локального хранения, API-моки для внешних сервисов).

## Как запустить приложение

1. Склонируйте репозиторий на свой локальный компьютер:
   ```bash
   git clone https://github.com/https-github-com-TSU-step/Paycheck2Paycheck
   ```

2. Откройте проект в Android Studio (рекомендуется последняя стабильная версия).
3. Дождитесь завершения синхронизации Gradle.
4. Выберите эмулятор или подключенное физическое Android-устройство.
5. Нажмите кнопку Run (Shift + F10) для сборки и запуска приложения.

## Реализованный сценарий и как его проверить

В текущей версии реализован главный пользовательский сценарий: заполнение бюджета, автоматический расчет и отображение дневного лимита на основном экране.

**Шаги для проверки сценария:**

1. Запустите приложение.
2. На главном экране нажмите на иконку шестеренки, чтобы перейти на отдельный экран настройки бюджета.
3. Введите общую доступную сумму и выберите дату окончания периода.
4. Подтвердите ввод и вернитесь на главный экран.
5. Убедитесь, что на главном экране корректно отображается рассчитанный дневной лимит, а также общий остаток средств и прогресс текущего периода.

## Где находятся моки и как они работают

Вся внутренняя бизнес-логика приложения и математические вычисления (расчет дневного бюджета, работа с датами и суммами) реализованы по-настоящему.

Моки используются исключительно для симуляции работы внешнего API распознавания голоса. Поскольку реального сервера для обработки аудио в MVP нет, функционал голосового ввода трат работает через заглушку в слое данных (Data Layer). При попытке записать трату голосом система не отправляет аудиофайл в сеть, а возвращает заранее подготовленный (замоканный) результат с распознанной суммой и категорией, имитируя успешный ответ от сервера.
